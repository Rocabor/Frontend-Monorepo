## 🎯 Overview

This repository is a **monorepo** documenting my frontend development journey through [Frontend Mentor](https://www.frontendmentor.io/) challenges. Each project is independently built and showcases progressive skill development.

### ✨ Features
- 📦 **23 completed challenges** across 3 difficulty levels
- 🎨 **Responsive designs** with mobile-first approach
- 🔗 **Live demos** for each project
- 📚 **Organized codebase** by difficulty

### 🖥️ Run Locally

Since many projects are built with **Vite** and **Tailwind CSS**, they require a build step to run properly. Follow these steps based on the project type:

#### 📁 For Vanilla HTML/CSS/JS Projects (no build required)
```bash
# Clone the repository
git clone https://github.com/Rocabor/Frontend-Monorepo.git
cd Frontend-Monorepo

# Navigate to any project folder
cd newbie/qr-code-component  # Example

# Simply open index.html in your browser
# On macOS:
open index.html
# On Windows:
start index.html
# On Linux:
xdg-open index.html
```

#### ⚡ For Vite + Tailwind Projects (build required)
```bash
# Clone the repository
git clone https://github.com/Rocabor/Frontend-Monorepo.git
cd Frontend-Monorepo

# Navigate to a Vite-powered project
cd newbie/results-summary-component  # Example with Vite + Tailwind

# Install dependencies
npm install

# Start the development server
npm run dev

The project will open automatically at http://localhost:5173 (or another port if 5173 is in use). The server will hot-reload as you make changes.
```

### 📋 Quick Reference
Guidelines for exploring and running the projects:


| Project Type |  How to Run |
| :--- | :--- |
| **Static HTML/CSS** | Open `index.html` directly in browser |
| **Vanilla JS** |  Open `index.html` directly in browser |
| **Vite + Tailwind** | `npm install` + `npm run dev` |


### 📦 Build for Production (optional)
If you want to build any Vite project for production, follow these steps:

   ```bash
  npm run build
  ```
The built files will be in the dist folder and can be deployed to any static hosting service.

```
💡 Tip:

Each project folder contains its own package.json with specific scripts and dependencies.

Check the "Technologies" column in the Project Index to know which setup a project requires.

All projects have live demos, so you can see them running without any local setup!
```
