Users should be able to:

- View the optimal layout for the site depending on their device's screen size
- See hover states for all interactive elements on the page
- Receive an error message when the `form` is submitted if:
  - Any `input` field is empty. The message for this error should say *"[Field Name] cannot be empty"*
  - The email address is not formatted correctly (i.e. a correct email address should have this structure: `name@host.tld`). The message for this error should say *"Looks like this is not an email"*

## My process

- Tailwind CSS v4 - Utility-first CSS framework
- Vanilla JavaScript

### What I learned

During this project, I learned to implement custom form validation using vanilla JavaScript, including real-time feedback and dynamic `aria-invalid` attributes for improved accessibility. I also explored how to effectively integrate SVG error icons within form input fields and refined my understanding of semantic HTML5 by using tags like `<article>` for autonomous content blocks.

### Continued development

I want to continue improving in:

- Exploring more advanced Tailwind v4 features, such as `container queries` and custom animations.
- Implementing more complex real-time form validation patterns.
- Enhancing keyboard navigation and overall accessibility for interactive components.
