### What I learned

This project helped me practice several modern CSS techniques:

- *1. Tailwind CSS v4 with custom utilities*
- *2. Fluid typography with clamp()*
- *3. Custom component styles using CSS layers*
- *4. Responsive images with picture element*
- *5. Gradient hover states*



### Continued development

This project highlighted several areas I want to continue focusing on in future challenges:

**1. Tailwind CSS v4 Custom Utilities** - I enjoyed creating custom utilities like `@utility preset-1` and `@utility cardshadow`. I want to deepen my understanding of Tailwind's new CSS-first configuration approach and learn to build more complex, reusable utility patterns.

**2. Fluid Typography with `clamp()`** - Using `clamp()` for responsive typography worked beautifully across all breakpoints. I plan to refine this technique further, perhaps creating a systematic set of fluid scale utilities that can be reused across different projects.

**3. CSS Layers Organization** - Separating styles into `@layer base`, `@layer components`, and `@layer utilities` kept the codebase clean. I want to explore more sophisticated layer strategies for larger projects, including proper cascade control and third-party integration.

**4. Responsive Image Strategies** - The `picture` element with multiple `source` tags and WebP fallbacks proved effective. I'd like to learn more about modern image formats (AVIF), lazy loading optimization, and art direction techniques.

**5. Accessible Interactive States** - I included `:focus` states alongside `:hover` for keyboard navigation. Moving forward, I want to ensure all interactive elements have proper focus management and ARIA labels, especially for complex components like cards with multiple interactive elements.

**6. Gradient Hover Effects** - The semi-transparent overlay technique for gradient buttons was interesting. I'd like to explore more creative hover transitions, including subtle animations and transform effects that enhance UX without being distracting.
