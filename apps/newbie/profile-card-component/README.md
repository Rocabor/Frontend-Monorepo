## My process

### Built with

- **Semantic HTML5 markup**
- **Tailwind CSS v4** - For utility-first styling and responsive design
- **Custom CSS with `@layer`, `@theme`, and `@utility` directives** - To extend Tailwind's capabilities
- **Mobile-first workflow** - Using Tailwind's variant system (`md:`, `xl:`)
- **Flexbox** - For card layout and content positioning
- **CSS Multiple Backgrounds** - For the decorative pattern elements

### What I learned

This project was a great opportunity to dive deeper into Tailwind CSS v4 and its new features, while also practicing complex CSS background positioning.

1. **Advanced Tailwind CSS v4 Configuration:**
   - Using `@layer base` to define global styles while maintaining Tailwind's organization
   - Creating custom utilities with `@utility` for consistent typography presets
   - Extending the theme with `@theme` for project-specific colors and shadows


2. **Complex Background Positioning:**

    - Learned to work with multiple background images and position them precisely
    - Used viewport units (vw/vh) for responsive background positioning
    - Implemented different background positions for various screen sizes

3. **Responsive Design with Variants:**

    - Leveraged Tailwind's @variant directive for responsive background adjustments
    - Used mobile-first approach with min-width breakpoints

   
### Continued development

  In future projects, I want to focus on:

  - Exploring CSS Grid for more complex layouts alongside Flexbox
  - Improving accessibility with better ARIA labels and keyboard navigation
  - Mastering Tailwind's JIT engine for even more efficient styling
